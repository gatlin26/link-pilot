# 标签参考内容系统设计

## 问题分析

### 当前情况
1. ✅ 工具有 `tool_references` 表存储参考内容
2. ❌ 标签没有对应的参考内容表
3. ❌ AI 生成标签定义时缺少参考依据

### 需求
标签系统需要参考内容来：
1. 生成准确的标签定义（name, description, content）
2. 为 AI 提供上下文信息
3. 支持人工审核和编辑

## 解决方案

### 方案 A：创建标签参考内容表（推荐）

#### 1. 数据库表设计

```typescript
// src/db/schema.ts

/**
 * 标签参考内容表 - 存储标签的参考信息
 * 用于 AI 生成标签定义和内容
 */
export const tagReferences = pgTable("tag_references", {
  id: text("id").primaryKey(),
  slug: text("slug").notNull().references(() => toolTags.slug, { onDelete: 'cascade' }),

  // 参考来源
  source: text("source").notNull(),  // 'wikipedia' | 'industry-standard' | 'manual' | 'ai-generated'
  sourceUrl: text("source_url"),     // 参考来源 URL

  // 参考内容（多语言）
  locale: text("locale").notNull(),  // 'en' | 'zh'
  referenceName: text("reference_name"),           // 参考名称
  referenceDescription: text("reference_description"),  // 参考描述
  referenceContent: text("reference_content"),     // 详细参考内容（Markdown）

  // 元数据
  confidence: integer("confidence"),  // 置信度 (0-100)
  verified: boolean("verified").default(false),  // 是否已人工验证
  verifiedBy: text("verified_by"),   // 验证人
  verifiedAt: timestamp("verified_at"),

  createdAt: timestamp("created_at").notNull().defaultNow(),
  updatedAt: timestamp("updated_at").notNull().defaultNow(),
}, (table) => ({
  tagReferencesSlugIdx: index("tag_references_slug_idx").on(table.slug),
  tagReferencesLocaleIdx: index("tag_references_locale_idx").on(table.locale),
  tagReferencesSlugLocaleIdx: index("tag_references_slug_locale_idx").on(table.slug, table.locale),
}));
```

## 实施方案

### 推荐：方案 B（配置文件 + 参考信息）

#### 优点
1. ✅ 不需要数据库迁移
2. ✅ 配置即代码，易于版本控制
3. ✅ 可以直接在代码中引用
4. ✅ 便于批量处理

#### 实施步骤

##### 1. 创建参考内容抓取脚本

```typescript
// src/scripts/fetch-tag-references.ts

import Anthropic from '@anthropic-ai/sdk';
import { getAllWhitelistTags, getTagCategory } from '@/config/tag-whitelist';

/**
 * 从维基百科或其他来源抓取标签参考内容
 */
async function fetchTagReferences() {
  const allTags = getAllWhitelistTags();
  const references: Record<string, any> = {};

  for (const slug of allTags) {
    const category = getTagCategory(slug);

    // 1. 尝试从维基百科获取
    const wikiContent = await fetchFromWikipedia(slug);

    // 2. 如果没有，使用 AI 生成参考内容
    if (!wikiContent) {
      const aiContent = await generateReferenceWithAI(slug, category);
      references[slug] = aiContent;
    } else {
      references[slug] = wikiContent;
    }
  }

  return references;
}

async function fetchFromWikipedia(slug: string) {
  // 将 slug 转换为维基百科搜索词
  const searchTerm = slug.replace(/-/g, ' ');

  try {
    // 使用维基百科 API
    const response = await fetch(
      `https://en.wikipedia.org/w/api.php?action=query&format=json&prop=extracts&titles=${encodeURIComponent(searchTerm)}&exintro=1&explaintext=1`
    );
    const data = await response.json();

    // 提取内容
    const pages = data.query.pages;
    const pageId = Object.keys(pages)[0];

    if (pageId === '-1') {
      return null; // 未找到
    }

    return {
      source: 'wikipedia',
      sourceUrl: `https://en.wikipedia.org/wiki/${encodeURIComponent(searchTerm)}`,
      content: pages[pageId].extract
    };
  } catch (error) {
    console.error(`Failed to fetch from Wikipedia: ${slug}`, error);
    return null;
  }
}

async function generateReferenceWithAI(slug: string, category: string) {
  const anthropic = new Anthropic({
    apiKey: process.env.ANTHROPIC_API_KEY,
  });

  const prompt = \`你是一个专业的技术词典编辑。请为以下标签生成参考定义。

标签: \${slug}
分类: \${category}

要求：
1. 提供准确、专业的定义
2. 说明该标签在技术/商业领域的含义
3. 如果可能，提供行业标准定义
4. 内容应该客观、中立

返回 JSON 格式：
{
  "source": "ai-generated",
  "content": "详细的参考内容（200-300 字）"
}\`;

  const message = await anthropic.messages.create({
    model: 'claude-opus-4-6',
    max_tokens: 1000,
    messages: [{ role: 'user', content: prompt }],
  });

  const content = message.content[0];
  if (content.type !== 'text') {
    throw new Error('Unexpected response type');
  }

  const jsonMatch = content.text.match(/\{[\s\S]*\}/);
  if (!jsonMatch) {
    throw new Error('Failed to extract JSON');
  }

  return JSON.parse(jsonMatch[0]);
}
```

## 参考内容的优先级

1. **维基百科** - 最高优先级，权威、客观
2. **行业标准定义** - 专业术语、商业概念
3. **手动输入** - 特殊标签、自定义标签
4. **AI 生成** - 兜底方案，基于上下文推理

## 数据流程

```
1. 抓取参考内容
   ├─ 维基百科 API
   ├─ 行业标准网站
   └─ AI 生成（兜底）
   ↓
2. 存储参考内容
   ├─ 配置文件（references 字段）
   └─ 或数据库表（tag_references）
   ↓
3. 基于参考生成定义
   ├─ AI 读取参考内容
   ├─ 生成 name 和 description
   └─ 生成详细 content
   ↓
4. 导入到数据库
   └─ tool_tag_translations 表
```

## 总结

### 推荐方案
使用**配置文件 + 参考信息**的方式：
1. 先抓取参考内容（维基百科 + AI）
2. 基于参考内容生成标签定义
3. 将参考信息和定义一起存储在配置文件中
4. 导入到数据库

### 优势
- ✅ 有据可依：所有定义都基于参考内容
- ✅ 可追溯：知道定义的来源
- ✅ 可验证：人工可以审核参考内容
- ✅ 可维护：修改参考内容即可重新生成定义
