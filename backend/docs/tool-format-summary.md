## 主要问题

### 1. 缺少必需章节

大部分中文版本的工具文件缺少以下必需章节：
- **What** (工具介绍) - 最常见的问题
- **Features** (功能特点) - 第二常见的问题
- **How** (使用方法) - 部分文件缺少

### 2. 章节顺序问题

部分文件的章节顺序不符合标准格式：
- 标准顺序：What → Features → How → Price → Tips → FAQ
- 部分文件章节顺序混乱

### 3. 章节内容为空

少数文件的某些章节只有标题，没有实际内容。

## 解决方案

### 方案 1: 自动修复（推荐）

使用脚本自动修复格式问题：

```bash
# 预览修复内容（不实际修改文件）
npm run tools:format:dry-run

# 自动修复格式问题
npm run tools:format:fix
```

### 方案 2: 手动修复

参考 `docs/tool-content-format.md` 文档，手动修复每个文件：

1. 确保包含所有必需章节（What、Features、How）
2. 按照标准顺序排列章节
3. 确保每个章节都有实际内容
4. 统一标题格式

## 标准格式参考

### 英文格式

```markdown
### What is [Tool Name]?
[工具介绍内容]

### Features of [Tool Name]
- [功能点 1]
- [功能点 2]

### How to Use [Tool Name]
1. [步骤 1]
2. [步骤 2]

### Pricing
[价格信息]

### Helpful Tips
- [提示 1]

### Frequently Asked Questions

#### [问题 1]
[答案 1]
```

### 中文格式

```markdown
### [工具名称]是什么？
[工具介绍内容]

### [工具名称]的功能
- [功能点 1]
- [功能点 2]

### 如何使用[工具名称]
1. [步骤 1]
2. [步骤 2]

### 价格
[价格信息]

### 有用的提示
- [提示 1]

### 常见问题解答

#### [问题 1]
[答案 1]
```

## 注意事项

⚠️ **重要**: 自动修复脚本会：
- 统一标题格式
- 重新排列章节顺序
- 但**不会**添加缺失的内容

如果文件缺少必需章节，需要：
1. 从英文版本翻译内容
2. 或者手动编写中文内容
3. 确保中英文版本内容一致
