Link Pilot · 项目理解导览

先把它看成一条流水线，
你就不会在代码里迷路。

Link Pilot 不是一个单独页面，而是一组浏览器扩展模块协同工作的系统：先采集外链机会，再把结果存起来，然后根据当前页面做智能匹配，最后把网站资料和评论自动填进表单里。

一句话理解

这是一个把 外链采集 → 本地存储 → 智能匹配 → 一键填表 串起来的浏览器扩展系统。你可以把它理解成：先建立“可投递机会库”，再在真实网页里自动找到最合适的机会并完成表单填写。

采集 从 Ahrefs 页面截获外链数据
匹配 根据当前页面推荐对应外链
填表 自动把资料写入真实网页表单

看源码时抓住这 4 个问题

如果你读着读着开始迷路，就回到这 4 个问题：谁发起流程、谁调度流程、谁在页面里真正执行、谁负责把结果存下来。

01. 核心总流程图

先看信息怎么在系统里流动，再往下看每一步细节。

Step A

用户页面

用户浏览普通网页，看到扩展按钮或打开侧边栏。

Step B

Content Script

贴在网页现场，负责收消息、读 DOM、执行表单操作。

Step C

Background

像总调度中心，负责开 Ahrefs、路由消息、组织流程。

Step D

Storage

像本地数据库，存网站资料、外链库、会话、提交记录。

Step E

Side Panel

像操作台，负责展示匹配结果、选择网站资料、点击填表。

Step F

回到页面

最终仍然由 Content Script 在真实网页中完成检测、填充与提交记录。

02. 五个阶段拆解

把整个项目压缩成 5 个容易记住的阶段。

1

进入页面，扩展先感知上下文

扩展先判断当前页面是什么、有没有表单、当前 URL 是什么。这一步还没有真正“填”，只是先弄清楚现场情况。

  • 判断页面是否能填表
  • 同步当前标签页 URL 和标题
  • 为后面的智能匹配准备上下文
pages/side-panel/src/SidePanelView.tsx
pages/content/src/handlers/message-handler.ts
2

采集外链数据

用户手动触发采集后，background 会打开 Ahrefs 页面，命令 Ahrefs 页里的脚本启动拦截器，把接口返回的外链数据拿出来。

  • 打开 Ahrefs Backlink Checker
  • 等待 content script 就绪
  • 拦截 API 响应并解析外链
chrome-extension/src/background/collection-manager.ts
pages/content/src/handlers/message-handler.ts
3

写入本地存储

采集回来的结果不会只在内存里停留。系统会去重、转成统一业务对象，然后写入机会库、外链库和批次记录。

  • Opportunity：后续可跟进机会
  • Managed Backlink：外链资源库
  • Collection Batch：本次采集记录
packages/storage/lib/base/base.ts
chrome-extension/src/background/collection-manager.ts
4

智能匹配推荐

当你打开某个网页时，系统会把当前 URL、域名、路径、关键词和外链库里的数据做比较，给出“最可能对应哪条外链”的推荐。

  • 推荐最佳匹配
  • 给出置信度
  • 生成备选结果
pages/side-panel/src/SidePanelView.tsx
chrome-extension/src/background/message-router.ts
5

一键填表并记录提交

用户选好网站资料后，content script 会去真实网页里识别表单字段，填入姓名、邮箱、网址、评论，并在成功后记录提交状态。

  • 表单检测
  • 自动填充与自动提交
  • 提交历史去重和台账记录

03. 四个核心角色

把这些模块拟人化后，你会更容易理解它们为什么分开存在。

现场执行员

Content Script

它贴在真实网页里，能直接看到页面 DOM，也能真正去点按钮、填输入框、读取页面内容。

  • 处理页面现场逻辑
  • 执行表单检测和自动填充
  • 在 Ahrefs 页接收拦截指令
总调度中心

Background

它不直接碰页面，但负责把整个流程串起来：接消息、开标签页、等待结果、分发消息、统一落库。

  • 消息路由
  • 采集流程编排
  • 跨组件协同
共享记忆库

Storage

它负责把关键状态和结果保存下来，所以系统才能记住你的网站资料、外链库、当前会话和提交记录。

  • 保存业务数据
  • 提供 live update
  • 支撑统计与回显
操作台

Side Panel

它是用户最直观看到的控制界面，负责展示推荐、切换网站资料、触发一键填表。

  • 展示当前页面状态
  • 显示智能匹配结果
  • 承接用户操作

05. 最后你应该怎么理解这个项目

下面这些结论，比背诵文件名更重要。

核心心智模型

  • 这不是一个单页应用，而是一个多运行时协作的浏览器扩展系统。
  • 采集和填表是两条主链路：前者负责找机会，后者负责使用机会。
  • 真正和网页打交道的是 Content Script；真正把流程串起来的是 Background。
  • Storage 的存在让“当前状态、历史记录、推荐结果”能够跨界面共享。
  • 当你读源码卡住时，先问自己：这段代码是在发起流程、调度流程、执行动作，还是保存结果？
如果你后面要继续深挖，建议先从 collection-manager、message-handler、SidePanelView 这 3 个入口看起。
